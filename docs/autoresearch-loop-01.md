# Autoresearch Loop 01

## Repo-style mapping
Following the autoresearch repo pattern as closely as practical for website work:

- `program.md` = human-owned operating brief
- target artifact = `src/pages/HvacDatabaseReactivation.tsx`
- neighboring context = `src/pages/AiAutomationHvac.tsx`
- evaluation = rubric in `program.md`
- keep/discard = explicit after each pass

## Why this page first
The HVAC database reactivation page is the clearest first candidate because:
- it already exists
- it has strong paid-traffic intent
- it has a nearby broad comparison page to help evaluate intent separation
- it is narrow enough to improve in a disciplined loop

## Loop 01 boundary
Only iterate on the paid HVAC database-reactivation page first.
Do not spread work across multiple pages until the loop produces a clearly better baseline.

## Success definition
A kept revision should make the page:
- clearer
- more trustworthy
- more specific to paid database-reactivation traffic
- less overlapping with the broad HVAC page

## Failure definition
Reject revisions that:
- add unsupported quantitative claims
- make the page sound more generic
- reduce distinction from the broad HVAC page
- create more clutter than clarity
